### Financial aid buildout

Merit aid likelihood and FAFSA/CSS Profile timeline tagging are shipped. The consent/UX pass this needs is also shipped: `FinancialAidConsentSection.tsx` (mounted on Profile) collects income bracket (ranges, not exact figures), state, and family size behind an explicit opt-in checkbox with plain-language "never required" framing; persisted via nullable `financial_aid_info_consent/income_bracket/state/family_size/updated_at` columns (migration `069_financial_aid_consent.sql`, applied). Unchecking consent clears the stored values.

- [x] **Net price calculator equivalent** across matched schools — `NET_PRICE_ESTIMATE_PROMPT` (`lib/anthropic.ts`) + `lib/net-price-estimate.ts` produce a rough estimated net-price RANGE per school from income bracket/state/family size plus real College Scorecard sticker-price/avg-net-price data where available (`lib/college-scorecard.ts`). Cached in `financial_aid_net_price_cache` (migration `077_net_price_estimate_cache.sql`, applied), keyed by school + income bracket + family size + state — never by student. Served via `POST /api/net-price-estimate`, hard-gated on `financial_aid_info_consent` + a real (non-"prefer_not_to_say") income bracket; students without that data see an inviting "add your financial info" prompt (`NetPriceEstimateSection.tsx`) on the School Detail page's Info tab, never a broken/blank calculator. Always shown with a "rough estimate, not a guarantee" disclaimer and a link out to the school's own official Net Price Calculator as the authoritative source.
- [x] **Scholarship ↔ aid-gap coalescence** — same `/api/net-price-estimate` response joins the estimate against `getFitTier`/`getMatchFactors` (`lib/scholarships.ts`), surfacing up to 5 Strong-Fit/Possible tracked scholarships as "could help close this gap" right under the estimate on School Detail — no duplicate second surface on the Scholarships page.
- [x] **Affordability as a real matches factor** — `schoolMatchingPrompt`'s new AFFORDABILITY SIGNAL section reasons about income bracket/state/family size (when consent + real data are present) as a genuine factor in `why_text` (e.g. flagging a likely financial stretch at a high-sticker reach/target, or a likely-affordable in-state public safety) — never invents a dollar figure, never activates for a student without real opted-in data, and doesn't change behavior at all for students who left it blank. `matches/generate/route.ts` builds this signal server-side and only includes it in the prompt when genuinely present. `PROMPT_VERSION` bumped to `2026-07-23b`.

First-gen / low-income framing care applies throughout: never gate core product value behind financial-data entry, keep it clearly optional, and pair any estimate with plain-language context (what "net price" actually means vs. sticker price) rather than assuming financial literacy.

Not built (explicitly out of scope per this task's own ground rules): no loan/debt calculator, no exact-dollar storage anywhere in the new cache table (only ranges), no Net Price Calculator API integration (none exists across schools) — the estimate is AI-reasoned from public sticker-price/avg-net-price data plus the bracket/state/family-size inputs, and is always presented as an estimate, not an official figure.
